# Smart Home Simulation

## Overview

This project implements a simulation of a smart home environment in Java. It models interactions between residents, pets, devices, and environmental factors over time. The system is event-driven and tracks device usage, resource consumption, and behavioral patterns within the household.

The simulation is designed to demonstrate object-oriented design principles and the use of common software design patterns in a complex, dynamic system.

## Features

- Simulation of a hierarchical house structure (house, floors, rooms)
- Autonomous agents (people and pets) interacting with devices
- Multiple device types with realistic behavior and state transitions
- Time-based simulation with periodic updates
- Event generation and logging system
- Resource consumption tracking (electricity, water, gas)
- Reporting system for activity, configuration, and consumption analysis
- Environmental simulation including temperature changes and sensor reactions

---

## Architecture

The system is built using object-oriented principles and incorporates several design patterns:

- **State Pattern** – managing device behavior (e.g., lights, dishwasher, car)
- **Factory Pattern** – creating device instances
- **Singleton Pattern** – centralized time management
- **Observer Pattern** – temperature sensors reacting to changes
- **Flyweight Pattern** – efficient handling of shared device state

## Core Components

### House Model

- `House` – main container of the simulation  
- `Floor`, `Room` – hierarchical structure  
- Rooms contain devices, people, and pets  

### Entities

- `Person` – resident capable of interacting with devices and generating events  
- `Pet` – simpler agent with limited interactions  
- `TemperatureEntity` – simulates environmental changes  

### Devices

All devices extend `AbstractDevice` and support:

- Power state management (on/off)
- Time-based actions
- Resource consumption tracking

Examples include:

- Lights  
- Heating panels  
- Washing machines and dishwashers  
- Refrigerator  
- Blinds  
- Car  
- Pet feeder  

### Simulation Engine

- `TimeSimulation` – entry point  
- `EventGeneratorTask` – drives the simulation loop  
- `TimeEntitySingleton` – manages simulation time and updates  

### Event System

- `Event` represents any interaction in the system  
- Generated by entities, devices, or environmental changes  
- Logged for further processing  

### Logging and Reporting

- `SimulationLogger` – records events and system activity  
- `Reports` – generates:
  - Event reports  
  - Consumption reports  
  - Activity summaries  
  - House configuration overview  

---

## How It Works

1. The simulation initializes the house structure, devices, and inhabitants.  
2. Time progresses in discrete steps.  
3. During each step:
   - Entities perform actions  
   - Devices react based on state and input  
   - Environmental conditions change  
4. Events are generated and logged.  
5. Reports are produced after the simulation completes.
